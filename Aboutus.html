Example

The interface of a music app

Here is a photo of a mobile music player interface:
  1. Visibility of system status: The interface clearly shows the currently playing song, the artist, album artwork, playback controls, and the progress of the current track.
  2. Match between system and the real world: The playback controls (play, pause, skip forward/backward) match the real world.
  3. User control and freedom: The user has full control over playback with clearly buttons to play, pause and skip.
  4. Consistency and standards: The layout and functionality of this interface is consistent with other widely used music player apps, making it familiar and easy to use.
There's a heuristics that not obey:
  1. Help users recognize and recover from errors:When a music file is damaged or unplayable, the page simply greys out the song name and is not responsive when clicked on, without prompting the user as to what the music is unplayable for.

Interface recommendations

Set the alert text after clicking on a playback error or file error to help the user find out what's wrong and prompts the user on how to fix it. To achieve Help users recognize and recover from errors
